================================================================================
  v5.00 構想：様々な工程表に対応する汎用化（2026-06-03 策定／土台 v5.00・v5.01 実装済み）
================================================================================

  ┌──────────────────────────────────────────────────────────────────────────┐
  │ 背景・目的                                                                │
  └──────────────────────────────────────────────────────────────────────────┘

    社内で運用している各種工程表（例: 2026年4月分「内管工事工程表」「本管工事工程表」）
    を本ツールで作成・代替できるようにする。ユーザーが提示した参考PDFは、ガント部・概要欄
    のみを参照（見出し・押印欄は対象外）。
    現行ガントの概要欄は「名称／概要／担当」の固定3フィールドで、工程表ごとに異なる
    列構成・項目名や、カテゴリ色分け・凡例・連結マイルストーン・旬目盛り等に未対応。
    これらを汎用化し、「概要列を自由に追加・挿入できる」「項目名を編集できる」等を実現する。

  ┌──────────────────────────────────────────────────────────────────────────┐
  │ 参考工程表の構造分析（ガント部・概要欄／要点）                          │
  └──────────────────────────────────────────────────────────────────────────┘

    ■ 内管工事工程表
      概要欄: No / 工事名称(地区＋物件名の2行) / 発注者 / 工種(2行可) / 用途 / 進捗(%) /
              担当者(色文字) / 施工業者(色文字) / 着工予定 / 完成予定・開栓予定(1セル2段)
      ガント: 施工業者で色分けしたバー＋「開栓予定」を小四角マーカーで別表示。
              月表示に旬(10/20日)目盛り。凡例＝施工業者5種＋開栓予定。

    ■ 本管工事工程表
      概要欄: No / 工事名称(2行) / 工種 / 口径及び延長(2行可) / 獲得件数(対象-目標-受注) /
              担当者(色文字) / 施工業者(色文字) / 着工予定 / 完成予定
      ガント: 施工業者で色分けしたバー＋「着手●→設計期限◆→発注期限◆」の連結
              マイルストーン(担当者色の矢印線)。旬目盛り。凡例＝担当者4名(色矢印線)＋
              施工業者バー＋マイルストーン3種。

  ┌──────────────────────────────────────────────────────────────────────────┐
  │ 現行ツールとのギャップ                                                    │
  └──────────────────────────────────────────────────────────────────────────┘

    ・任意の列構成・項目名編集 … 未対応（固定3フィールド：名称/概要/担当）
    ・複数行セル … 概要のみ（他列は不可）
    ・カテゴリ列による色分け＋凡例 … 未対応（色は手動指定のみ・凡例なし）
    ・副マーカー／連結マイルストーン … 単発◆のみ（着手→設計→発注の連結なし）
    ・旬(10日)目盛り … 未対応（スケールは 日／週／月）

  ┌──────────────────────────────────────────────────────────────────────────┐
  │ 設計方針（核）                                                            │
  └──────────────────────────────────────────────────────────────────────────┘

    「概要欄の汎用カラムモデル化」を全拡張の土台とする。現行の固定3フィールドを、
    ユーザー定義の列配列 S.gantt.columns に置き換える。各タスクの値は node.cols[colId]
    に保持。後方互換として既存 text/desc/assignee を既定列へ自動移行（旧ファイルも開ける）。
    設計原則（単一HTML・サーバー不要・オフライン完結）は厳守。マーカー・凡例・旬目盛りは
    この土台の上に段階追加する。

  ┌──────────────────────────────────────────────────────────────────────────┐
  │ 要件一覧（R1〜R6）                                                        │
  └──────────────────────────────────────────────────────────────────────────┘

    R1. 概要欄の汎用カラムモデル
        - S.gantt.columns=[{id,title,type,width,align,lines,colorKey?,…}]、
          値は node.cols[colId]。
        - 列の 追加／挿入／削除／並べ替え／幅変更／項目名(見出し)編集。
        - 列ヘッダー行の描画（現状は見出し無し）。
        - 列型: text / multiline / index(No.自動) / date / percent / category。
    R2. 複数行セル（工事名称・口径及び延長 等の2行表示の一般化）。
    R3. カテゴリ列による自動色分け＋凡例自動生成
        - 任意の category 列（施工業者／担当者 等）を「色キー」に指定 → 値ごとに色割当
          → バー色＋凡例ボックスを自動生成。
    R4. マーカー／マイルストーン拡張
        - 主バーと別の予定マーカー（開栓予定 など）。
        - 連結マイルストーン列（着手→設計期限→発注期限 を担当者色の矢印線で連結）。
    R5. 時間軸の旬(10日)目盛り付き月表示（実物の見た目に合わせる）。
    R6. 工程表テンプレート（内管／本管 等のプリセット＝列構成・凡例の雛形保存）。

  ┌──────────────────────────────────────────────────────────────────────────┐
  │ 段階的開発計画（案・運用ルールの小刻みリリース準拠）                    │
  └──────────────────────────────────────────────────────────────────────────┘

    ── 第一段階 確定仕様（2026-06-03 ユーザーすり合わせで確定）──
    描画は「縦積み維持＋ユーザーが列を足せる」方式に確定（横並びのスプレッドシート表には
    “しない”）。版戦略=v5.00 整数繰り上げ／適用範囲=ガント専用。

    ◆ 左情報欄の新構成（1タスク＝4行）
      行1 タスク名称 … 全幅（列で割らない・結合風／名称として編集）
      行2 工程       … 全幅（列で割らない／日付として編集）
      行3・行4: 列ごとの自由セル「項目名：本文」
        - ⊕（情報欄と工程バーの境目付近）で右に1列追加＝全タスク共通で列が増える。
          各セルの項目名・本文はタスクごとに自由。直近入力の項目名を記憶し初期値に。
        - 既定の1列目: 行3＝概要／行4＝担当（既存 desc/assignee を引き継ぐ・編集可）。
        - 列幅は境目ドラッグ（⇔）で調整。折返し＋ぶら下げを維持し、幅変更で自動再折返し
          （行3・行4は折返し行数ぶん高さ可変）。
      ◆ 値の持ち方: 組込（名称/工程/概要/担当）は既存ノードフィールドに bind。新規追加列のみ
        node.cols に格納。startDate/endDate は一次データのまま（バー計算無改修）。旧ファイルは
        columns 欠落時に既定列を合成して無損失で開く。

    ◆ 同時に入れる改修
      ① 完了日: 表示範囲に「完了日」を追加し時間軸を開始日～完了日で区切る（S.gantt.endDate）。
      ② 完了日で変わるのは“文字だけ”: 右端からはみ出すタスクバーのラベルを右詰めに自動切替
         （列幅・レイアウトは動かさない）。
      ③ 情報欄へのクリック／ドロップでもタスク等を追加（現状は左欄クリックは無視）。
      ④ 新規タスク等は左クリックしたタスクの直下の行に挿入（無選択時は最下行）。

    ── 段階的開発計画（運用ルールの小刻みリリース準拠）──
    v5.00  第一段階（上記「確定仕様」）＝縦積み列モデル土台＋⊕列追加＋列幅ドラッグ
                    ＋①完了日／②右詰め文字／③クリック・ドロップ追加／④直下挿入。
                    big-bang（整数繰り上げ）。内部は段階的に実装し各段で Claude_Preview 検証。
    v5.01〜 列編集の拡充: 列の削除／並べ替え、型付け（index/percent/category 等）、複数行セル一般化。
    v5.0x  G3 カテゴリ色分け＋凡例自動生成（色キー列）。
    v5.0x  G4 マーカー拡張: 副マーカー＋連結マイルストーン。
    v5.0x  G5 旬(10日)目盛り。
    v5.0x  G6 仕上げ: 工程表テンプレート（雛形の任意提示・任意適用）。
    ※ R1〜R6（前掲）は引き続き全体の指針。第一段階は R1（列の自由化）＋R2（複数行セル）の
      中核を“縦積み維持”の形で先取りする。

  ┌──────────────────────────────────────────────────────────────────────────┐
  │ 検討事項・未確定                                                          │
  └──────────────────────────────────────────────────────────────────────────┘

    ・【設計思想・確定 2026-06-03】本ガントの設計思想は「このガントチャート様式を
      ベースにすれば、ほとんどの工程表を作れる」こと＝汎用・柔軟な土台の維持。
      参考工程表(内管/本管)の特定様式を“忠実に模倣”すると柔軟性が失われるため、個別様式の
      再現それ自体は目的としない（＝採用しない要素＝様式のハードコード／固定テンプレ強制）。
      → R1〜R6 は「特定工程表の複製」ではなく、「汎用の積み木（自由な列・色キー・マーカー・
        目盛り）でユーザーが各自の工程表を組み立てられる」方向で設計する。
      ※ R6『工程表テンプレート』は“雛形の任意提示・任意適用”に留め、強制様式にはしない
        （参考工程表は到達可能性を確認するための題材であって、再現対象ではない）。
    ・【確定 2026-06-03】版戦略＝v5.00 整数繰り上げ（big-bang 土台）。
    ・【確定 2026-06-03】適用範囲＝ガント専用（他図種は対象外）。
    ・【確定 2026-06-03】描画＝縦積み維持＋ユーザーが列を足す方式（横並び表にはしない）。
    ・後方互換: 旧版(v4.x)のガント（text/desc/assignee/phaseId 等）を無損失で開く
      （columns 欠落時に既定列を合成。組込列は既存フィールドに bind）。
    ・色キー/凡例の配置（図内 or 余白）と印刷時の扱い（v4.30 の改ページ・倍率との整合）。【後段】
    ・連結マイルストーンのデータモデル（タスクに複数の予定点を持たせる方式）。【後段】

  │ 先行実施済みの関連改善（v4.30・詳細は開発記録.txt）                     │

    v5.00 構想に先立ち、ガントの基礎品質・出力を v4.30 で改善済み:
      ・タスクバー文字の自動縮小／収まらない時はシェイプ上に左揃え、マイルストーン文字は
        シェイプ上に中央揃え。
      ・概要のぶら下がりインデント、空欄プレースホルダ撤去、編集セルのカーソルを通常に。
      ・PDF出力: 用紙サイズ/向きの自由選択、倍率(%)指定、フェーズ境界で改ページ、図面名を
        出力左上に表示。
      ・フェーズ行背景をフェーズ色の薄塗りに、縦バンドのハンドルを選択時のみ表示、
        カスタム色ピッカー＋フェーズ濃色プリセット。
    これらは v5.00（汎用カラムモデル）の前提となる土台整備でもある。
