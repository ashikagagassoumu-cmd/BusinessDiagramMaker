  v8.01（2026-07-01）複数選択・水平垂直ドラッグ・レーン色・デザインテンプレート・おしゃれステップ図
  ──────────────────────────────────────────────────────────────────────────────

  背景・目的:
    ①選択とドラッグの操作性を一般的な作図ツール（PowerPoint 等）に合わせたい、②レーン色を
    後から手軽に変えたい、③図の「色は変えずに見た目（角丸・線・影）だけ」を一括で整えたい、
    ④画像で示された番号サークル/矢羽/ピルのようなインフォグラフィック調のステップ図を作りたい、
    というユーザー要望。①〜③は明確なので即実装、④は見た目主体のため事前にHTML/SVGモックで
    複数案を提示し、採用テンプレを決めてから実装した。

  設計上の判断（落とし穴と対策）:
    ・修飾キーの役割を「押した瞬間」に確定させない。従来は Ctrl+mousedown で即複製していたため、
      Ctrl+クリック＝複数選択と両立できなかった。対策＝mousedown では通常ドラッグを仕込むだけにし、
      「クリック確定(mouseup・未移動)＝選択トグル」「ドラッグ確定(mousemove・4px超)＝Ctrlでコピー化
      (convertDragToCopy)」に振り分けた。convertDragToCopy を必ず S_didDrag 成立後に呼ぶ点が肝
      （指の微揺れで誤コピーしないため）。Ctrl/Shift どちらのクリックでも複数選択できる。
    ・Shift+ドラッグの水平/垂直固定は、mousemove 冒頭で pt を制約（変位の大きい軸だけ残す）すると
      下流の全分岐（group / 自由配置 / レーン）に一括で効く。ただし整列・グリッド・レーン内吸着が
      固定軸を動かしてしまうので、S.drag.constrained のとき該当軸のスナップを無効化して軸を純粋に
      固定する。
    ・デザインテンプレート（S.style / STYLES）は「色（各シェイプの fill/stroke・レーンの bg/hd）は
      一切変えず、形の装飾（角丸rx倍率・線幅・影フィルタ・アクセント帯・矢羽/ピル形状）だけ」を
      差し替える方針。shapeEl の rx を styRx()、線幅を stySW() 経由にし、影とアクセント帯は
      renderNodes で shapeEl の戻り要素へ後付け（テキストに影を掛けない）。矢羽/ピルは基本の
      「プロセス(rect)」の形のみ差し替え（他 rect 系は角丸だけ変化）＝ガント等を壊さないため。
      影は SVG フィルタ（ds-*）を defs に静的定義。保存/復元に S.style・S.stepNums を追加。
    ・おしゃれステップ図: 番号サークルは新シェイプ step_num（円＋大きな01/02/03＋下にラベル）。
      番号は「読み順（横広がりなら左→右／縦広がりなら上→下）」で毎描画時に自動採番（stepOrder）
      するため、並べ替え・追加で番号が自動で振り直る。番号バッジ(S.stepNums)は一般のプロセス系
      シェイプ左上にも同ロジックで付与。配色テーマは作成済みシェイプへ色を上書きする明示操作
      （Ctrl+Z で戻せる）＝既定方針「色は変えない」の任意的な例外として提供。レイアウト挿入は
      既存の addNode/addEdge 相当で編集可能なノード列＋接続線を生成（挿入後もドラッグ/文字編集可）。

  既知の注意:
    ・デザインテンプレートと配色テーマは図全体（全図種）に効く（S.style はグローバル）。矢羽テンプレは
      「プロセス(rect)」のみ形が変わる。step_num は自由配置シェイプ扱い（レーンに拘束されない）。

  追補（v8.01 同セッション・版番据置）:
    ・レイアウト挿入を縦横モード追従に変更。挿入時 isV() を見て、縦モード＝上→下の列／横モード＝
      左→右の行で並べる。挿入後に向きを切り替えても toggleOrientation の全ノード x↔y 入替で列ごと
      回転する（番号は stepOrder が読み順を再判定して追従、番号サークルのラベルは縦並び＝円の右／
      横並び＝円の下へ _flowDirs で自動切替）。
    ・ステップ数を可変に。挿入した列を flowId で1グループとして管理し、列末尾に＋（flowAddStep＝末尾へ
      1つ追加・並び方向を継続）／－（flowRemoveStep＝末尾を1つ削除・最低1つは残す）ボタンを描画
      （renderFlowControls）。ボタンは選択モード時のみ表示し、class="flow-ctrl" を付けて
      buildExportSvg のクローンから除去＝SVG/PNG/PDF には出さない。


  背景・目的:
    建設・土木の施工工程表（バーチャート式）は、用紙上部に 工事名・工事場所・発注者・
    施工者・工期・担当（現場代理人等）・作成日・版数 を載せ、右上に承認（作成・確認・承認の
    押印）欄、下部に凡例を置くのが定番。これまでガントは時間軸ヘッダー(2段)しか無く実務の
    工程表として提出しづらかった。Web調査で項目名・配置の定番を確定した上で実装。現場ごとに
    記載項目が異なるため「自由な行（ラベル：値）の追加・編集・並べ替え」でカスタマイズ可能に。

  設計上の最重要判断（落とし穴と対策）:
    画面の当たり判定は全箇所 (clientY - rect.top)/zoomLevel のピクセル換算で、SVGの viewBox
    オフセットを考慮していない（例: クリック無視ガード if(py<GANTT_HEADER_H)、ganttSlotIndexAtY）。
    そのため「エクスポートで既に使っている負Y方向 viewBox 帯」を画面にも使うとクリック位置が
    全部ズレる。
    → 採用方式: 見出しを “上部の非操作ストリップ（正Y空間 y:0..ganttHeadH()）” として描き、
      時間軸ヘッダー・グリッド・行を見出し高さぶん下げる。下げ量を ganttBodyTop()
      （= ganttHeadH() + GANTT_HEADER_H）に集約し、「内容の上端＝GANTT_HEADER_H」だった参照を
      これに一括置換。時間軸ヘッダー(renderGanttHeader)は内部座標を一切変えず translate グループで
      平行移動（DOMイベント＋svgPt はそのまま効くので＋ボタン・列幅リサイズも不変）。
      これらをライブSVGへ描くので buildExportSvg のクローンにそのまま乗り、SVG/PDF出力が自動追従。
      画面の座標系は従来どおり「ユーザー単位＝ピクセル・原点0」を維持＝当たり判定は上端定数の
      差し替えのみで済む。

  レイアウト確定（モック反復でユーザーと合意）:
    工事名（大）＋情報を左に2列・下詰め（本体との余白を詰める）／右上に作成日＋承認欄（小さめ・
    画面の並びは左→右で承認・確認・作成＝紙面では右から作成・確認・承認）／凡例は「押印欄の下・
    右側ゾーン内」で横長（3列×2段）・角丸の黒枠で囲む（説明欄の下には侵入させない）。
    見出しの高さは行数・承認ON/OFF・凡例項目数で自動可変（ganttHeadModel が縦寸法のみ算出、
    横位置は描画時に SVG_W から右端基準で決定）。

  データモデル / 永続化 / 後方互換:
    S.gantt.header = {show,title,rows:[{label,value}],showDate,approval:{show,slots:[承認,確認,作成]}}
    S.gantt.legend = {show,auto,overrides:{key→ラベル上書き},extra:[{label,color,stroke}]}
    approval.slots は「描画順＝左→右」で格納（紙面で右から作成のため、表示時に右寄せ配置）。
    保存は saveHtmlCore が S.gantt 全体を JSON 化するので追加フィールドも自動で永続化される。
    旧ファイル（header/legend 無し）は描画前に ganttEnsureHeader() が既定補完＝そのまま開ける。

  凡例の自動生成:
    ganttLegendItems() が S.nodes を走査し、使われている要素だけを拾う＝タスク（予定）／進捗
    （progress>0 のとき）／マイルストーン／フェーズ／休工（suspensions 有り）／土日。色見本は
    SD.gantt_* の fill/stroke を参照（図と完全一致）。overrides でラベルだけ差し替え、extra で
    「電気設備（協力会社）」等の工種・業者色分けを手動追加できる。

  エクスポート反映:
    ・SVG/PNG: ライブSVGに描くのでクローンに自動で乗る。見出し表示時は buildExportSvg の図面名
      タイトル帯を省略（工事名と二重化しないため）。
    ・PDF: 表題＋時間軸ヘッダーを <thead> に内包し各ページ先頭で繰り返す（viewBox を 0..ganttBodyTop()
      に拡張）。本体はフェーズ境界で改ページ（従来どおり）。図面名帯は見出し表示時は出さない。
    ・Excel: 帯(BAND)を「1 + 見出し情報行数」に拡張し、行1に工事名＋作成日、続く行に情報2列、
      （承認ONなら）承認欄をテキストで展開。最下部に凡例（色塗りセル＋ラベル）を1行追加。
      freeze/printTitlesRow は時間軸のみ繰り返す既存挙動を踏襲（情報帯は1ページ目先頭のみ）。

  既知の注意:
    ・見出し表示時は calcCanvasSize で SVG_W を最低 1120px 確保（狭い図でも右ゾーン＝承認欄＋凡例が
      重ならないように）。
    ・設定UIの動的リスト（行・承認・凡例）はテキスト入力にフォーカス中は作り直さない（ganttHeadEditing
      ガード）＝入力中のキャレット/フォーカスを維持しつつ SVG だけライブ更新。

  検証（Claude_Preview 実DOM ＋ node --check）:
    headH＝承認OFF:164／ON:176、行数で追従。bodyTop=232 でクリック→正しいタスクindex、
    header OFF で bodyTop=56（＝完全に従来挙動・回帰なし）。設定UI＝行の追加/削除/▲▼並べ替え・
    承認トグル・凡例ラベル上書き・工種色分け追加が即反映。保存→JSON→復元ラウンドトリップで
    header/legend 保持。旧形式（header/legend 無し）も補完して描画。buildExportSvg は工事名を内包し
    図面名帯の二重化なし（nameBand=0）。PDF/Excel いずれも例外0で生成（Excel 13.8KB 出力確認）。
    全インラインスクリプトの node --check ＝SYNTAX_OK。

  （diagram_v7.11.html をコピーして diagram_v8.00.html を新規作成。<title>／#app-title-bar／serve.js 既定
    配信先を v8.00 に更新。大変更＝整数部を v8 に。次の小変更は v8.01。）

    検証（Claude_Preview 実DOM）: 削除＝ホバー表示/確認後に列とcells破棄・c0は非対象。表示＝

■ 左サイドバー（編集バー）にスクロールを追加
  why: 見出し・凡例パネルが縦に長く（工事場所等の情報行＋承認欄＋凡例）、画面が低いと
       下側のコントロール（承認欄・凡例設定など）が画面外に隠れて操作不能になっていた。
  how: #app-header（図の名称欄）は固定のまま、その下の全セクション（シェイプ〜レーン）を
       #sidebar-scroll でくくり、flex:1 + min-height:0 + overflow-y:auto で縦スクロール化。
       ヘッダーは常時見える＝図の名称をいつでも編集できる。

■ 見出しの「工事名」を図の名称に一本化（見出し専用の工事名入力を廃止）
  why: 「工事名」と「図の名称」を二重入力させていた。実質同じものなので二度手間・不整合の元。
  how: 見出しパネルの工事名 input（gh-title）を撤去。renderGanttTitleBlock と Excel 出力の
       工事名は exportDiagramName()（＝S_diagramName＝上部「図の名称」）を参照。図の名称欄の
       oninput を onDiagramNameInput() に変更し、ガント＋見出し表示中は即 render() で
       タイトルをライブ追従。h.title は不参照（旧保存データの title は無視・後方互換で実害なし）。
  検証: Claude_Preview 実DOMで、図の名称編集→見出しタイトルが即時更新／工事名入力欄が消滅／
        サイドバーが scrollHeight>clientHeight でスクロール可／console エラー0 を確認。

【v8.02】スイムレーン等の接続線を改良（ラベル線上化・○ホバー化・終点整形・段違い） 2026-07-01
--------------------------------------------------------------------------------
■ 発端（ユーザー指摘・4点）
  ① 接続線のラベル/編集欄が線上に出ない（端点の直線中点＝折れ線から浮く）。
  ② ラベル無し線の○（追加ボタン）が常時表示でうるさい。
  ③ 1つの相手に加えて2つ目へ繋ぐと、扇状分散で1本目の始点がずれてL字に曲がる（真下なのに斜行）。
  ④ 3つ目へ繋ぐと2本目と3本目の横線が同じ高さで重なる。

■ 仕様決定（AskUserQuestion で合意・モックで反復）
  ・始点(出口)は等間隔(扇状分散)のまま許容。終点は「一直線にできる線は一直線を優先(端が中央で
    なくても可)／できない線は辺の中央に着地」。相手シェイプごとに判定を変える。
  ・重なり回避は「曲がる高さ(横トラック)を枝ごとに段違い」。ただし素直に段違いにすると外側の枝が
    内側の枝の横線を突っ切る＝クロスが出る（ユーザーが不可）。
  ・角丸など線の描き方は現行維持（今回いじるのは経路の座標＝frac と段違い位置だけ）。

■ クロスを出さない段違いの原理（重要・why）
  出口辺を共有する“曲がる線”を、相手までの距離が【近い相手＝相手寄り(下段)／遠い相手＝始点寄り
  (上段)】のトラックに置く。こうすると各線の縦の降り口は自分のトラックまでしか下りず、遠い相手の
  長い横線は近い相手の短い横線より上を通り、互いの縦線が相手の横線を突っ切らない＝交差も重なりも
  出ない（源が箱幅より広く相手が横に散る配置でも成立）。逆順(遠い=下段)にすると必ず交差する。

■ 実装（diagram_v8.02.html）
  ・allocatePorts 末尾に段違いパスを追加（['swimlane','flowchart','bpmn'] のみ対象）。
    出口辺ごとにグループ化→各線の出口X/Yを算出→終点 entryFrac を直線化(範囲内なら exitに一致)／

【v8.06】2026-07-21 — ガント「時（1時間ごと）」モード追加
================================================================================
■ 目的・経緯（why）
  ・ユーザー要望「ガントに1時間当たりの工程モードを追加。ただし既定は従来どおり1日モード」。
  ・仕様は着手前にモック提示＋質問で確定：24時間通し／複数日OK／1時間刻み（分は常に:00）。
    → 稼働時間帯の絞り込みや夜間スキップは行わず「連続タイムライン」を採用。日モードの比例配置
      ロジックをそのまま1時間解像度へ落とすだけで済み、実装リスクを最小化できるため。

■ 設計判断（how）
  ・日時の持ち方: タスク/マイルストーンの startDate/endDate/date を、日モード等は従来の
    `YYYY-MM-DD`、時モードでは `YYYY-MM-DDTHH:00` に拡張。時刻なしは 00:00 とみなす（後方互換）。
    → S.gantt に新フィールドを足さず scale:'hour' だけで表現。snap/restore/init/import は
      既存の Object.assign(defaultGantt(),…) 経由で自動 round-trip、旧ファイルも無改変で開ける。
  ・コアは ganttParseDate の正規表現に任意の時刻部を追加し、書き込み側だけ scale 対応の新関数
    ganttFmt(d,scale) を通す（ganttFmtDate は日付のみのまま据置＝日付入力欄・週/月原点整形を壊さない）。
    ganttAddUnit/ganttDiffUnit に hour 分岐、ganttXToDate を ganttFmt 化。
  ・終了時刻は「排他」に決定（例 9:00〜12:00＝3時間ぶん・12:00 の格子で終端）。日モードの
    「終了日+1日（包含）」とは意味論が異なるため、calcGanttPositions・フェーズ帯・Excelバーの
    3か所で時モードのみ +0（排他）に分岐。モックで提示した見え方（9〜12＝3枠）と一致させた。
  ・編集ダイアログは時モードのときだけ日付入力を datetime-local(step=3600) に切替え、
    保存時に分を :00 へ丸める（ganttInputVal / ganttNormInput）。
  ・Excel: サブ列構築に hour 枝（1時間＝1列・上段=日付/下段=時）。colDateIdx は時刻対応 parse で自動追従。

■ 既知の制約（注意）
  ・休工期間(suspensions)は日単位のまま（編集は type=date を維持、内部の +1日 計算も据置）。
    時モードでも休工は「日」粒度で穴が空く。実害は限定的と判断しスコープ外とした。
  ・既定の完了日は月末(日付)のため、時モードに切替えると 24h×日数ぶん（最大数百列）に広がる。
    重い場合は「完了日」を絞る運用。Excel はサブ列 CAP=500 で保護。

■ 検証（how tested）
  ・追加パッケージ不要（npm レジストリはポリシーで 403）。プリインストール Chromium を
    CDP + Node22 組込み WebSocket で駆動し実機確認：
    既定=日モード／時へ切替(colW=24)／9:00〜12:00 タスクが3列幅で描画／ヘッダー上段=日付・下段=時／
    ganttXToDate 往復一致・時差=3／編集ダイアログが datetime-local／日モードへ戻して回帰なし／
    Excel 出力(時モード)が例外なく完了。全工程で console エラー 0。

【v8.06 追補】2026-07-21 — 印刷(PDF)の時間軸ヘッダーが2ページ目以降に出ない不具合を修正
────────────────────────────────────────────────────────────────────────────────
■ 症状: ガント/工程表の「印刷」出力で、日付・時間の時間軸ヘッダーが1ページ目にしか出ず、
  2ページ目以降はタスク行だけになる。時/日/週/月すべてのスケールで再現（縦方向のページ分割の
  問題のためスケールに依存しない。v8.06 の新機能が原因ではなく従来からの印刷処理のバグ）。
■ 原因(why): 印刷用HTMLは工程表ヘッダーを <thead>(display:table-header-group) に入れて各ページ
  先頭で繰り返す設計。ところが本体(タスク行)を <tbody> の巨大な単一 <tr> にしていたため、
  1つの <tr> が用紙高さを超えて分割されると <thead> が再描画されない（thead の繰り返しは
  「行と行の境目」でしか起きず、単一行の途中改ページには効かない）。フェーズ無しの図では全タスクが
  1 <tr> になり必ずこの状態になる。
■ 対策(how): expPDF のガント分岐を再構成。
  ①各ページ先頭で繰り返すのは「時間軸＋列見出し」だけに限定（表題部＝工事名/図面名帯は pg-once で
    表の外側に一度だけ描き、1ページ目のみ表示）。ユーザー確定＝2ページ目以降は時間軸のみでよい。
  ②本体を「フェーズのまとまり＝1<tr>」または「フェーズ外タスク＝1ページ相当の高さごとのチャンク＝1<tr>」に
    分割。改ページが必ず行境界で起きるようになり、<thead> が全ページ先頭で再描画される。
    ※各 <tr> は SVG 全体を1回直列化するため、当初「1タスク=1行」にしたら直列化がタスク数の二乗コストになり
      printToPDF が固まった（実ブラウザ印刷でも同様）。そこでチャンク高さ maxChunkPx=(W/倍率)×0.7 でまとめ、
      直列化回数をページ数程度に抑えた（W=チャート幅・倍率＝印刷倍率。横長チャートはチャンク大＝ページ少、
      行が支配的なら小＝ページ毎に分割されヘッダー反復。0.7=A系横向きの縦横比を下限に）。
  ③フェーズの切れ目では pg-break(page-break-before) で必ず改ページ（フェーズは途中で分割しない。
    フェーズ名の見た目を保つ。ユーザー確定）。末尾の空き行は従来どおり印刷しない。
■ 既知の注意: 1フェーズが1ページ高さを超える稀なケースは、そのフェーズの2ページ目にヘッダーが
  出ない（フェーズを途中分割しない方針の副作用として許容）。ヘッダーは全幅を用紙幅に合わせる関係で
  縦に圧縮されるが、これはページ1も同条件（従来からの挙動。必要なら印刷倍率を上げる）。
■ 検証(how tested): npm 不可のためプリインストール Chromium を CDP+Node22 WebSocket で駆動し、印刷用HTMLから
  実 PDF を生成→PDFビューアでページ2以降を目視。フェーズ無し(時/日/週/月)＝全ページ先頭に時間軸ヘッダーが
  反復・表題部は1ページ目のみ。フェーズ有り＝各フェーズが自ページ始まり・ヘッダー反復・フェーズ色帯/名は保持。
  thead スライスを単体描画し濃色帯＋日付＋時刻(0〜23)が正しいことも確認。印刷HTMLは0.4〜0.6MBで固まらない。

【v8.07】2026-07-21 — ガント休日グレーアウトの刷新（時モード夜間帯／土日祝チェック／祝日自動判定）＋2改善
────────────────────────────────────────────────────────────────────────────────
■ 背景・要望（why）
  ・時（1時間）モードでも土日祝が「終日」グレーで、稼働時間帯まで塗り潰され実態に合わない。
    → 時モードは夜間 20:00〜翌6:00 のみを非稼働として薄グレーにしたい（毎日）。
  ・休日は現場ごとに異なるため、土/日/祝を選べるようにしたい。ONにした曜日/祝日は全モードで終日グレー。
  ・追加要望2件: ①時モードの初日を必ず 0:00 起点で表示。②グレーアウトでフェーズ帯の色が途切れる。

  ・グレーアウトの二層化: 「夜間帯(night)」＝時モードのみ・全日 20-6・薄トーン(GANTT_SHADE_NIGHT #f3f6f9)。

  ・祝日は内蔵ロジックで自動判定（外部取得なし＝単一HTML原則を維持）。固定祝日＋ハッピーマンデー(第n月曜)＋
    春分/秋分(概算式・1980〜2099有効)＋振替休日(日曜祝日の連鎖の翌非祝日)＋国民の休日(祝日に挟まれた平日)。
    ラベル赤字は ganttIsRedLabel＝「土日 or 祝日」で常時（チェック非依存＝事実としての休日表示）。
  ・週/月モードは1列が粗い(週=2日刻みExcel／画面は週/月列)ため、画面グレーアウトは従来どおり非対象。
    Excel は既存の日ベース細分化(v6.07)に相乗り＝土日/祝の終日＋時モードは夜間帯も塗り分け(full/night)。
  ・UI はユーザー指定によりサイドバーでなく上部ツールバー(#gantt-tools)の「表示」右隣に土/日/祝チェックを新設
    （モックで配置・挙動を事前提示し承認済み）。ganttSetHoliday でトグル→render、updateGanttToolbar で同期。
■ 追加改善の how
  ・時モード初日0:00: ganttAxisOrigin() の hour 分岐で開始日の時刻を切り捨て（ganttFmtDate で当日00:00を原点に）。
  ・フェーズ帯の途切れ: フェーズ・ヘッダー行の時間軸部分は最背面(g-lanes/renderGanttRows)に薄フェーズ色で
    塗るため、その上の g-grid のグレーアウトで色が抜けていた。renderGanttGrid 末尾でフェーズ行の時間軸部分を

■ 既知の制約・注意
  ・週/月モードの画面グレーアウトは列粒度が粗いため非対応（従来どおり）。休日チェックの効果は日/時で確認できる。
  ・祝日の概算式(春分/秋分)は 1980〜2099 年で有効。天皇誕生日は現行(2/23・2020〜)のみ対応（過去の12/23は非対応）。
■ 検証（how tested）

【v8.07 追補】2026-07-22 — 印刷(PDF)のページ分割を「フェーズ毎に必ず改ページ」→「途切れる時だけ改ページ」に戻す
────────────────────────────────────────────────────────────────────────────────
■ 経緯（why）: v8.06追補で入れた「フェーズの切れ目で必ず改ページ(pg-break=page-break-before:always)」が
  ユーザーの意図と異なっていた。本来は「基本は極力1ページにまとめ、フェーズがページ境界で途切れる場合だけ改ページ」
  （＝従来仕様）が望ましい。ユーザー指示によりバグ修正として直接 main へ反映（版番据置の追補）。
■ 対策（how）: 2段構え。
  (a) expPDF のフェーズ<tr>への強制改ページ(pg-break=page-break-before:always)を廃止（phaseBreak を false 固定）
      ＝フェーズ毎に必ず改ページするのをやめ、基本は1ページにまとめる。
  (b) ただし pg-break を外しただけでは、1フェーズが用紙高さを超えると<tr>が途中で分割され、<thead>の日付
      ヘッダーが2ページ目以降で反復されず消える（v8.06追補の元バグと同じ構造。当初の追補はここが不完全だった）。
      そこで flushPhase を修正し、フェーズの<tr>も maxChunkPx(≒用紙1ページ)ごとに分割 emit（while ループ）＝
      全<tr>が用紙高さ以下になる。
  結果、強制改ページはせず（page-break-inside:avoid でフェーズ/チャンクがページ境界で途切れる時だけ改ページ）、
  かつ各<tr>が用紙以下なので <thead>(table-header-group)が全ページ先頭で反復し日付ヘッダーが消えない。
  <thead>の全ページ反復・チャンク分割・表題部1ページ目のみ(pg-once)は v8.06追補のまま。未使用化した CSS
  ルール .pg-break は残置（無害・コメントで不使用と明記）。
■ 検証（how tested）: 構文チェック(script 2本・エラー0)。プリインストール Chromium を CDP+Node22 WebSocket で
  駆動し印刷HTML/PDFを生成して確認：thead に日付ヘッダー(SVG height=56)を含む／大フェーズ(タスク50)で本体が
  6ブロックに分割され各<tr>高さ≤1584(=maxChunkPx・用紙1ページ相当)以下／pg-break 付与0／--print-to-pdf で
  2ページPDF生成。各<tr>が用紙以下のため <thead> が全ページ反復＝日付ヘッダーは全ページに出る。
  ※headless環境に pdftoppm 不在のため PDF の画素目視は不可。構造・高さ・ページ数で保証（実機の印刷プレビュー推奨）。
