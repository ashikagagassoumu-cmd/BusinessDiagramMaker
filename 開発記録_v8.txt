  v8.01（2026-07-01）複数選択・水平垂直ドラッグ・レーン色・デザインテンプレート・おしゃれステップ図
  ──────────────────────────────────────────────────────────────────────────────

  背景・目的:
    ①選択とドラッグの操作性を一般的な作図ツール（PowerPoint 等）に合わせたい、②レーン色を
    後から手軽に変えたい、③図の「色は変えずに見た目（角丸・線・影）だけ」を一括で整えたい、
    ④画像で示された番号サークル/矢羽/ピルのようなインフォグラフィック調のステップ図を作りたい、
    というユーザー要望。①〜③は明確なので即実装、④は見た目主体のため事前にHTML/SVGモックで
    複数案を提示し、採用テンプレを決めてから実装した。

  設計上の判断（落とし穴と対策）:
    ・修飾キーの役割を「押した瞬間」に確定させない。従来は Ctrl+mousedown で即複製していたため、
      Ctrl+クリック＝複数選択と両立できなかった。対策＝mousedown では通常ドラッグを仕込むだけにし、
      「クリック確定(mouseup・未移動)＝選択トグル」「ドラッグ確定(mousemove・4px超)＝Ctrlでコピー化
      (convertDragToCopy)」に振り分けた。convertDragToCopy を必ず S_didDrag 成立後に呼ぶ点が肝
      （指の微揺れで誤コピーしないため）。Ctrl/Shift どちらのクリックでも複数選択できる。
    ・Shift+ドラッグの水平/垂直固定は、mousemove 冒頭で pt を制約（変位の大きい軸だけ残す）すると
      下流の全分岐（group / 自由配置 / レーン）に一括で効く。ただし整列・グリッド・レーン内吸着が
      固定軸を動かしてしまうので、S.drag.constrained のとき該当軸のスナップを無効化して軸を純粋に
      固定する。
    ・デザインテンプレート（S.style / STYLES）は「色（各シェイプの fill/stroke・レーンの bg/hd）は
      一切変えず、形の装飾（角丸rx倍率・線幅・影フィルタ・アクセント帯・矢羽/ピル形状）だけ」を
      差し替える方針。shapeEl の rx を styRx()、線幅を stySW() 経由にし、影とアクセント帯は
      renderNodes で shapeEl の戻り要素へ後付け（テキストに影を掛けない）。矢羽/ピルは基本の
      「プロセス(rect)」の形のみ差し替え（他 rect 系は角丸だけ変化）＝ガント等を壊さないため。
      影は SVG フィルタ（ds-*）を defs に静的定義。保存/復元に S.style・S.stepNums を追加。
    ・おしゃれステップ図: 番号サークルは新シェイプ step_num（円＋大きな01/02/03＋下にラベル）。
      番号は「読み順（横広がりなら左→右／縦広がりなら上→下）」で毎描画時に自動採番（stepOrder）
      するため、並べ替え・追加で番号が自動で振り直る。番号バッジ(S.stepNums)は一般のプロセス系
      シェイプ左上にも同ロジックで付与。配色テーマは作成済みシェイプへ色を上書きする明示操作
      （Ctrl+Z で戻せる）＝既定方針「色は変えない」の任意的な例外として提供。レイアウト挿入は
      既存の addNode/addEdge 相当で編集可能なノード列＋接続線を生成（挿入後もドラッグ/文字編集可）。

  既知の注意:
    ・デザインテンプレートと配色テーマは図全体（全図種）に効く（S.style はグローバル）。矢羽テンプレは
      「プロセス(rect)」のみ形が変わる。step_num は自由配置シェイプ扱い（レーンに拘束されない）。


  背景・目的:
    建設・土木の施工工程表（バーチャート式）は、用紙上部に 工事名・工事場所・発注者・
    施工者・工期・担当（現場代理人等）・作成日・版数 を載せ、右上に承認（作成・確認・承認の
    押印）欄、下部に凡例を置くのが定番。これまでガントは時間軸ヘッダー(2段)しか無く実務の
    工程表として提出しづらかった。Web調査で項目名・配置の定番を確定した上で実装。現場ごとに
    記載項目が異なるため「自由な行（ラベル：値）の追加・編集・並べ替え」でカスタマイズ可能に。

  設計上の最重要判断（落とし穴と対策）:
    画面の当たり判定は全箇所 (clientY - rect.top)/zoomLevel のピクセル換算で、SVGの viewBox
    オフセットを考慮していない（例: クリック無視ガード if(py<GANTT_HEADER_H)、ganttSlotIndexAtY）。
    そのため「エクスポートで既に使っている負Y方向 viewBox 帯」を画面にも使うとクリック位置が
    全部ズレる。
    → 採用方式: 見出しを “上部の非操作ストリップ（正Y空間 y:0..ganttHeadH()）” として描き、
      時間軸ヘッダー・グリッド・行を見出し高さぶん下げる。下げ量を ganttBodyTop()
      （= ganttHeadH() + GANTT_HEADER_H）に集約し、「内容の上端＝GANTT_HEADER_H」だった参照を
      これに一括置換。時間軸ヘッダー(renderGanttHeader)は内部座標を一切変えず translate グループで
      平行移動（DOMイベント＋svgPt はそのまま効くので＋ボタン・列幅リサイズも不変）。
      これらをライブSVGへ描くので buildExportSvg のクローンにそのまま乗り、SVG/PDF出力が自動追従。
      画面の座標系は従来どおり「ユーザー単位＝ピクセル・原点0」を維持＝当たり判定は上端定数の
      差し替えのみで済む。

  レイアウト確定（モック反復でユーザーと合意）:
    工事名（大）＋情報を左に2列・下詰め（本体との余白を詰める）／右上に作成日＋承認欄（小さめ・
    画面の並びは左→右で承認・確認・作成＝紙面では右から作成・確認・承認）／凡例は「押印欄の下・
    右側ゾーン内」で横長（3列×2段）・角丸の黒枠で囲む（説明欄の下には侵入させない）。
    見出しの高さは行数・承認ON/OFF・凡例項目数で自動可変（ganttHeadModel が縦寸法のみ算出、
    横位置は描画時に SVG_W から右端基準で決定）。

  データモデル / 永続化 / 後方互換:
    S.gantt.header = {show,title,rows:[{label,value}],showDate,approval:{show,slots:[承認,確認,作成]}}
    S.gantt.legend = {show,auto,overrides:{key→ラベル上書き},extra:[{label,color,stroke}]}
    approval.slots は「描画順＝左→右」で格納（紙面で右から作成のため、表示時に右寄せ配置）。
    保存は saveHtmlCore が S.gantt 全体を JSON 化するので追加フィールドも自動で永続化される。
    旧ファイル（header/legend 無し）は描画前に ganttEnsureHeader() が既定補完＝そのまま開ける。

  凡例の自動生成:
    ganttLegendItems() が S.nodes を走査し、使われている要素だけを拾う＝タスク（予定）／進捗
    （progress>0 のとき）／マイルストーン／フェーズ／休工（suspensions 有り）／土日。色見本は
    SD.gantt_* の fill/stroke を参照（図と完全一致）。overrides でラベルだけ差し替え、extra で
    「電気設備（協力会社）」等の工種・業者色分けを手動追加できる。

  エクスポート反映:
    ・SVG/PNG: ライブSVGに描くのでクローンに自動で乗る。見出し表示時は buildExportSvg の図面名
      タイトル帯を省略（工事名と二重化しないため）。
    ・PDF: 表題＋時間軸ヘッダーを <thead> に内包し各ページ先頭で繰り返す（viewBox を 0..ganttBodyTop()
      に拡張）。本体はフェーズ境界で改ページ（従来どおり）。図面名帯は見出し表示時は出さない。
    ・Excel: 帯(BAND)を「1 + 見出し情報行数」に拡張し、行1に工事名＋作成日、続く行に情報2列、
      （承認ONなら）承認欄をテキストで展開。最下部に凡例（色塗りセル＋ラベル）を1行追加。
      freeze/printTitlesRow は時間軸のみ繰り返す既存挙動を踏襲（情報帯は1ページ目先頭のみ）。

  既知の注意:
    ・見出し表示時は calcCanvasSize で SVG_W を最低 1120px 確保（狭い図でも右ゾーン＝承認欄＋凡例が
      重ならないように）。
    ・設定UIの動的リスト（行・承認・凡例）はテキスト入力にフォーカス中は作り直さない（ganttHeadEditing
      ガード）＝入力中のキャレット/フォーカスを維持しつつ SVG だけライブ更新。

  検証（Claude_Preview 実DOM ＋ node --check）:
    headH＝承認OFF:164／ON:176、行数で追従。bodyTop=232 でクリック→正しいタスクindex、
    header OFF で bodyTop=56（＝完全に従来挙動・回帰なし）。設定UI＝行の追加/削除/▲▼並べ替え・
    承認トグル・凡例ラベル上書き・工種色分け追加が即反映。保存→JSON→復元ラウンドトリップで
    header/legend 保持。旧形式（header/legend 無し）も補完して描画。buildExportSvg は工事名を内包し
    図面名帯の二重化なし（nameBand=0）。PDF/Excel いずれも例外0で生成（Excel 13.8KB 出力確認）。
    全インラインスクリプトの node --check ＝SYNTAX_OK。

  （diagram_v7.11.html をコピーして diagram_v8.00.html を新規作成。<title>／#app-title-bar／serve.js 既定
    配信先を v8.00 に更新。大変更＝整数部を v8 に。次の小変更は v8.01。）

  v8.00 追補（2026-06-30）情報欄（概要列）の操作性改善＝列削除・項目名の表示/編集まわり
  ──────────────────────────────────────────────────────────────────────────────
    版番据置の追補（バグ修正＋微改善）。what はコミット参照。why／原因と対策のみ記す。

    ① 追加列を削除する手段が無かった → 列ヘッダー下端中央にホバー表示の削除アイコン（赤×）を
       追加。確認ダイアログ（showConfirm）で警告してから削除し、列構成と各タスクの cells を破棄。
       既定列 c0（概要/担当＝desc/assignee bind）は基幹のため対象外（誤渡しもガード）。
       アイコン位置はユーザー指示で「ヘッダー行の下端」に統一（＋列追加ボタンも下端へ）。

    ② バグ: 追加列のセルは「本文(値)が非空のときだけ描画」していたため、項目名(ラベル)だけ
       入力しても何も表示されなかった（ユーザー指摘＝編集しても出てこない）。
       → 描画条件を「本文が非空、または 追加列で項目名が非空」に変更。c0 は従来どおり本文が
         空なら既定ラベル(概要/担当)も非表示＝空タスクで「概要：/担当：」を並べない挙動を維持。

    ③ 項目名を「列単位（全タスク共通）」に変更（ユーザー選択＝B案）。従来は項目名をタスクごとに
       cell.l で保持していたため、列見出しとして不自然だった。col.r3Label/col.r4Label に集約し、
       ganttColLabel/ganttSetColLabel を新設、ganttCellLabel(n,…) は列ラベルを返す薄いラッパに。
       本文(値)は引き続きタスクごと（c0=desc/assignee, 追加列=cells[colId].v）。
       後方互換: 旧データ（タスク別 cell.l）は ganttMigrateColLabels() が列ラベルへ集約（最初の
       非空を採用・冪等）。読込・インポート・undo復元の各経路に挿入し既存ファイルの項目名を保全。

    検証（Claude_Preview 実DOM）: 削除＝ホバー表示/確認後に列とcells破棄・c0は非対象。表示＝
    項目名のみでも追加列は表示・c0空はラベル非表示。列単位＝1タスクの項目名変更が全タスクへ反映、
    本文はタスク別、旧データ移行で「工区」を列へ集約、undo/redoで列ラベル復元。コンソールエラー0。

【追補】v8.00（版番据置・バグ修正） 2026-06-29
--------------------------------------------------------------------------------
■ 左サイドバー（編集バー）にスクロールを追加
  why: 見出し・凡例パネルが縦に長く（工事場所等の情報行＋承認欄＋凡例）、画面が低いと
       下側のコントロール（承認欄・凡例設定など）が画面外に隠れて操作不能になっていた。
  how: #app-header（図の名称欄）は固定のまま、その下の全セクション（シェイプ〜レーン）を
       #sidebar-scroll でくくり、flex:1 + min-height:0 + overflow-y:auto で縦スクロール化。
       ヘッダーは常時見える＝図の名称をいつでも編集できる。

■ 見出しの「工事名」を図の名称に一本化（見出し専用の工事名入力を廃止）
  why: 「工事名」と「図の名称」を二重入力させていた。実質同じものなので二度手間・不整合の元。
  how: 見出しパネルの工事名 input（gh-title）を撤去。renderGanttTitleBlock と Excel 出力の
       工事名は exportDiagramName()（＝S_diagramName＝上部「図の名称」）を参照。図の名称欄の
       oninput を onDiagramNameInput() に変更し、ガント＋見出し表示中は即 render() で
       タイトルをライブ追従。h.title は不参照（旧保存データの title は無視・後方互換で実害なし）。
  検証: Claude_Preview 実DOMで、図の名称編集→見出しタイトルが即時更新／工事名入力欄が消滅／
        サイドバーが scrollHeight>clientHeight でスクロール可／console エラー0 を確認。

【追補】v8.00（版番据置・改善） 2026-06-30
--------------------------------------------------------------------------------
■ 見出し（工事名・工事場所など）の編集を「ダブルクリックで開くダイアログ」に変更
  why: 見出しの編集欄を左サイドバー（幅250px）に置いていたため、1行が「ラベル＋値＋×」で
       横に詰まり、長い住所・会社名が見切れて入力しづらかった（ユーザー指摘＝狭すぎる）。
       タスク詳細編集と同じ「対象をダブルクリック→ダイアログ」の作法に揃える方が一貫する。
  方式（ユーザー選択＝B案・モック合意済み）: 見出し領域を覆う透明ヒット矩形を
       renderGanttTitleBlock に追加し、ダブルクリックで #gantt-head-modal（幅560px）を開く。
       サイドバーからは入力リスト（情報行/承認スロット/凡例上書き）を撤去し、表示ON/OFF等の
       トグルと「見出しを編集…」ボタンだけ残す（ユーザー指示＝トグルのみ）。
  設計: データモデル（header.rows / approval.slots / legend）は不変＝PDF/Excel/SVG出力は無改修で
       追従。ダイアログは既存setter（ganttHeadSetRow等）を再利用しライブプレビュー。開いた時点の
       header/legend/図名称を _ghBackup に退避し、OK=snap()確定／閉じる(=Esc/背景)=破棄して復元。
       動的リストの作り直しはテキスト入力フォーカス中スキップ（ganttHeadDlgEditing）でキャレット保持。
       工事名はダイアログ内でも「図の名称」と双方向連動（ganttHeadDlgSetTitle）。
  検証: Claude_Preview 実DOMで、ダブルクリック/サイドバーボタンの両方で起動・工事名/行/承認/凡例の
       編集がSVGへ即反映・行の追加/削除/▲▼・承認/凡例トグルでスロット/上書き欄が増減・OK確定で
       snap・閉じるで全項目バックアップ復元・サイドバーはトグル5個＋編集ボタンのみ・console エラー0。
